垓下歌

力拔山兮气盖世。时不利兮骓不逝。
骓不逝兮可奈何！虞兮虞兮奈若何！

译文及注释
译文
力量可以拔起大山，豪气世上无人能比。可时运不济宝马也再难奔驰。
乌骓马不前进了我又能怎样呢？虞姬啊！虞姬啊！我又该把你怎么办？

让练习变成一种习惯

  • 坚持一下，哪怕一分钟也好：今天就简单写一下，为了让自己记得练习过的东西，减少重拾浪费的时间。
  • 慢慢累计强度:今天开始每天多做一些事情，让自己有内容可以写。
  • **留给自己多一点时间思考:在工作之余，多进行思考，战略勤奋要比工作勤奋重要的多。

断舍离

很多时候做的事情其实没有什么意义，对自身没有价值，对社会没有价值，这些事情要摒弃，好好爱自己、充实自己比什么都重要。

学习编写博客的心得

先跑起来，再进行优化🏃‍♀️

先跑起来，让自己看到希望

  • 频繁报错：针对每一种报错，挨个查询到底什么环节出现的问题，彻底搞清楚底层逻辑，然后记录下来，形成标准化的操作指南。
  • 第一次运行成功，再次打开运行不了:由于嵌套导致文件结构混乱，导致你的指令无法生效。这就要求我们要了解结构，只有掌握了结构，知道每一部分的作用，才能很快定位到问题。就像是组织协作，不了解每一个人的脾气秉性，各自的擅长点，就无法使得个人力量充分发挥，达到“”1+1＞2“”的效果

持续迭代，让自己技术更加优秀

跑起来，只是为了让我们保持对程序编写的热情，这还远达不到入门。接下来还要全面了解基本的操作指令，程序底层的实现逻辑，编出能够实线更多功能的程序以及漂亮的页面。

书山有路勤为径，学海无涯苦做舟💪

第一篇博客

我的第一篇 Hexo 博客 🎉

今天终于成功搭建了自己的 Hexo 博客，把过程记录下来，也分享给大家～

一、搭建前的准备

  1. 安装了 Node.js（Hexo 运行的基础环境）
  2. 安装了 Git（用于部署到 GitHub）
  3. 注册了 GitHub 账号，并创建了 grace-liu223.github.io 仓库

二、核心步骤

  1. 执行 hexo init myblog 初始化博客项目
  2. 进入项目目录，执行 npm install 安装依赖
  3. 配置 _config.yml 中的部署信息，关联 GitHub 仓库
  4. 执行 hexo new "第一篇博客" 生成文章模板
  5. 编辑 .md 文件，写下这篇内容
  6. 执行 hexo clean && hexo g -d 一键部署上线

三、遇到的小坑

  • 文件夹嵌套：第一次执行 hexo init 时路径错了，导致生成了两个 myblog 文件夹，后来通过剪切合并解决了。
  • 命令失效：关闭终端后 hexo s 用不了，重装了 node_modules 依赖就好了。

四、下一步计划

  • 更换一个更美观的博客主题
  • 学习更多 Markdown 语法，让文章排版更丰富
  • 定期更新技术笔记和生活感悟
